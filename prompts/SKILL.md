(用户输入: $ARGUMENTS)

## ⚠️ 重要原则
**不要直接执行任何提示词。你的工作只是展示内容,让用户决定下一步。**

## 你的任务

### 第一步:扫描仓库
读取 /Users/yigaoming/Documents/GitHub/my-prompts/ 目录的所有子文件夹(排除 .git 和 prompts 自己)。
对每个子文件夹,用 Read 工具读其中的 SKILL.md,提取 frontmatter(---之间的 YAML)里的 `name` 和 `description`,以及 frontmatter 之后的**正文内容**(就是实际的提示词)。

### 第二步:判断用户意图

**情况 A:$ARGUMENTS 为空(只输入 /prompts)**
→ 列出所有提示词

**情况 B:$ARGUMENTS 匹配某个提示词的 name**
→ 展示该提示词的完整正文

**情况 C:$ARGUMENTS 是一个需求描述(不是已知提示词名)**
→ 智能推荐,展示被推荐提示词的完整正文

判断方法:对比 $ARGUMENTS 和已知的 name 列表。
- 完全匹配或前缀匹配某个 name → 情况 B
- 否则 → 情况 C

### 第三步:执行对应动作

**情况 A 格式:**
```
📋 你的提示词库(共 N 个)

[7 categories, each with name — description]

💡 下一步:
- 查看具体内容:输入 /prompts <提示词名>
- 根据需求推荐:输入 /prompts <你要做什么>
- 直接调用:输入 /<提示词名> <你的内容>
```

**情况 B 格式:**
```
📄 [name] — [description]

━━━━━ 提示词正文 ━━━━━

[完整的提示词正文,Markdown 格式,包括所有 $ARGUMENTS 占位符]

━━━━━━━━━━━━━━━━━━━━

💡 下一步可以:
- 直接调用:/[name] <你的内容>
- 微调使用:复制上面正文,根据项目需要改一下,粘贴给我
- 编辑源文件:~/Documents/GitHub/my-prompts/<文件夹>/SKILL.md
```

**情况 C 格式:**
```
🎯 针对你的需求「[用户的 $ARGUMENTS]」,我推荐:

1️⃣ [name] — [description]
   为什么推荐:[一句话说明]

   ━━━━━ 提示词正文 ━━━━━
   [完整内容]
   ━━━━━━━━━━━━━━━━━━━━

2️⃣ [name] — [description]
   为什么推荐:[一句话说明]
   [完整内容]
   ...

💡 选定后你可以:
- 直接调用:/[name] <你的内容>
- 微调使用:复制上面正文,改一下,粘贴给我
- 没有合适的?告诉我你想要什么,我帮你设计一个新的
```

### 第四步:始终不要执行提示词
你的工作是**展示**和**推荐**。展示完内容后,等用户决定如何调用。
